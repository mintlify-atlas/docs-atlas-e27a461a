## Overview

`p-inlinemessage` is a compact, non-dismissible status chip intended for placement adjacent to form controls or inside layout cells. It is lighter than `p-message` — there is no close button and the padding derives from the shared form-field spacing tokens — making it ideal for inline validation feedback such as "Email is required" beneath an input.

## Usage

```html
<!-- Standalone inline messages -->
<p-inlineMessage severity="success" text="Username is available." />
<p-inlineMessage severity="info"    text="Password must be 8+ characters." />
<p-inlineMessage severity="warn"    text="Caps Lock is on." />
<p-inlineMessage severity="error"   text="Email address is invalid." />
```

### Paired with a Form Field

```html
<div class="flex flex-col gap-1">
  <label for="email">Email</label>
  <input
    pInputText
    id="email"
    [(ngModel)]="email"
    [ngClass]="{ 'ng-invalid ng-dirty': emailInvalid }"
  />
  <p-inlineMessage
    *ngIf="emailInvalid"
    severity="error"
    text="Enter a valid email address."
  />
</div>
```

### With Custom Icon

```html
<p-inlineMessage severity="info">
  <ng-template pTemplate="icon">
    <i class="pi pi-info-circle"></i>
  </ng-template>
  Sync runs every 15 minutes.
</p-inlineMessage>
```

## Design Tokens

Defined in `src/lib/theme/inlinemessage/index.ts`:

### Root

| Token | Value |
|---|---|
| `root.padding` | `{form.field.padding.y} {form.field.padding.x}` |
| `root.borderRadius` | `{content.border.radius}` |
| `root.gap` | `0.5rem` |

Padding inherits from the global form-field padding token so inline messages visually align with adjacent inputs.

### Text & Icon

| Token | Value |
|---|---|
| `text.fontWeight` | `500` |
| `icon.size` | `1rem` |

## Nettalco Semantic Color Mappings

| Severity | Semantic Token | Primitive Palette | Brand Hex (500) |
|---|---|---|---|
| `success` | `{success.*}` | `nettalcoSuccess` | `#2BA5CD` (Cyan) |
| `info` | `{info.*}` | `nettalcoInfo` | `#66A6FB` (Sky Blue) |
| `warn` | `{warn.*}` | `nettalcoWarn` | `#D57952` (Orange) |
| `error` | `{error.*}` | `red` (standard) | `#ef4444` (Red) |

Because `p-inlinemessage` shares the same semantic palette as `p-message` and `p-toast`, a `warn` inline message will always match the Nettalco Orange hue (`#D57952`) of a warn toast shown simultaneously, keeping the visual language consistent across all feedback surfaces.

> **Accessibility note:** Inline messages render as `role="alert"` or `role="status"` depending on severity. Pair with `aria-live` regions when used inside reactive forms to announce validation changes to screen readers.
